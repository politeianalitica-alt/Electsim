Pregunta 1 de 5 — Arquitectura de producto: ¿cuál es la capa primaria?

El diagnóstico muestra que el frontend es una maqueta de alta fidelidad con backend sólido. El siguiente bloque de trabajo debe definir qué capa construimos primero. Hay tres filosofías distintas — cada una implica una hoja de ruta diferente.

A
Recomendada
Object Graph + Canvas
El modo Palantir Gotham
Cada entidad es un objeto: actor, norma, narrativa, alerta, evento, documento. Navegar la plataforma es navegar el grafo. El canvas es el espacio donde el analista conecta objetos, crea hipótesis y produce entregables.
Pedro Sánchez → 3 iniciativas · 2 alertas · 7 narrativas
Ley Vivienda → 12 actores · 4 alertas · riesgo 81
Narrativa "lawfare" → 87 arts · 5 actores · subiendo
Canvas abierto: "Bloqueo Junts May 2026"
Valor diferencial
  • Máxima diferenciación vs clipping y dashboards
  • Flujo natural: detectar → investigar → producir
  • Crea dependencia: el usuario vive en el canvas
  • Surfacing real del backend semántico (pgvector)
Complejidad / riesgo
  • Más complejo de implementar (grafo + canvas)
  • Curva de adopción mayor para nuevos usuarios
B
Intelligence Feed + Tiempo Real
El modo Newsroom de élite
La pantalla principal es un feed semántico inteligente: briefing matinal, alertas en vivo, narrativas emergentes, pulso electoral. Todo llega en tiempo real. El usuario consume, prioriza y actúa desde el feed.
CRÍTICA TC admite recurso — hace 12 min
Narrativa vivienda +18% — acelerando
Briefing matinal actualizado — 08:02
IPC mejor de lo esperado — oportunidad
Valor diferencial
  • Primera pantalla del día → hábito inmediato
  • SSE/WebSocket: alerta en tiempo real
  • Más rápido de implementar (conectar APIs reales)
  • Familiar para usuarios de medios y clipping
Complejidad / riesgo
  • Menor diferenciación (parecido a Feedly avanzado)
  • Sin canvas, falta capa de producción de entregables
C
Workspace Operativo + Multi-tenant
El modo War Room de campaña
El workspace es el centro de gravedad: cada cliente tiene su sala de guerra con su agenda, sus issues, su equipo, sus decisiones y sus entregables. La inteligencia llega al workspace contextualizada para ese cliente.
Workspace: PP Campaña 2027
Issues críticos: 3 · Acciones pendientes: 12
Equipo activo: 4 analistas
Briefing personalizado para este cliente
Valor diferencial
  • Multi-tenant real: cada cliente en su sala
  • Colaboración en equipo nativa
  • Monetización clara: workspace = suscripción
  • Alineado con NationBuilder para campaña
Complejidad / riesgo
  • Requiere routing [workspaceId] completo
  • Sin object graph, falta profundidad analítica
Mi recomendación: A + B combinados — Object Graph como arquitectura de fondo, Intelligence Feed como punto de entrada diario. C se construye encima. Pero quiero saber cuál es el caso de uso que tienes más urgente de resolver antes de decidir el orden exacto de construcción.
